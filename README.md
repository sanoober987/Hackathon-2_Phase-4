# Cloud Native Todo Chatbot (Phase IV)

This project is part of the **GIAIC Cloud Native / AI Engineering Program**.

The goal of this phase is to deploy a **Todo Chatbot application** on a **local Kubernetes cluster using Minikube** with **Helm charts**, while using **AI-assisted DevOps tools** like kubectl-ai, Kagent, and Docker AI (Gordon).

The application was originally built in **Phase III** and then containerized and deployed using Kubernetes in **Phase IV**.

---

## 🚀 Project Overview

The Todo Chatbot allows users to manage tasks through a chat interface.  
In this phase, the focus is on **cloud-native deployment and container orchestration**.

Main objectives:

- Containerize frontend and backend using Docker
- Deploy services on Kubernetes (Minikube)
- Use Helm charts for application deployment
- Use AI DevOps tools for cluster management

---

## 🧰 Technology Stack

| Component | Technology |
|-----------|------------|
| Frontend | Next.js |
| Backend | FastAPI |
| Containerization | Docker (Docker Desktop) |
| Orchestration | Kubernetes (Minikube) |
| Package Manager | Helm |
| AI DevOps Tools | kubectl-ai, Kagent |
| Docker AI | Gordon |

---

## 📦 Containerization

The application consists of two main services:

- **Frontend Container**
- **Backend Container**

Docker was used to build and run the containers locally before deployment.
